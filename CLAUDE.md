# Perch — Project Rules for Claude Code

**プロダクト名**: Perch（パーチ）
**位置付け**: 揮発しない「賢いmi」＋Obsidian Vault昇格＋AI「あのあれ検索」
**形態**: Tauri 2 デスクトップアプリ（macOS）／完全ローカル
**バージョン**: v1.0（新規プロジェクト・2026-06-29）

このファイルは Claude Code が Perch プロジェクト全体で守るべき **絶対ルール**、**アーキテクチャ要約**、**ドキュメント参照ガイド** を定義する。各 Wave の実装時、エージェントは必ず本ファイルを参照すること。

> Git運用はプロジェクト非依存の正本 **GIT-WORKFLOW.md** に従う（本ファイルでは Git 運用を重複記載しない）。

## 🎯 プロジェクトの核心（絶対に見失わないこと）

Perch の存在意義は **「未保存メモの揮発を根絶する」** こと。ユーザーは mi でタブを大量展開し、PC 再起動（特に予期しない強制再起動）で未保存メモを失い、サルベージ作業を繰り返してきた。

- **揮発防止（データ堅牢性）は最優先。** ここがチープだとツール全体が使われなくなる。
- **設計思想：「賢いmiは完成品、AIは拡張スロット」。** 土台（賢いmi）は最初から100点で作る。AI 機能は差し込み口を用意しつつ、MVP に含むもの（あのあれ検索・タグ付け）はチープにしない。

## 🏗️ アーキテクチャ要約

| レイヤー | 採用 | 補足 |
|---------|------|------|
| アプリ形態 | Tauri 2（macOS デスクトップ） | Chromeと別プロセスで軽量常駐。Electron不採用（Chromium同梱で重く、"Chromeが重いから逃げたい"目的と矛盾） |
| フロントエンド | Next.js（静的書き出し）or Vite + React | Tauri内WebViewで描画 |
| バックエンド | Rust（Tauri標準・薄く使う） | ファイルアクセス等のOS機能のみ。ロジックの大半はJS/TS側 |
| エディタ | CodeMirror 6 | フォントサイズ変更・HTML/JS/CSSハイライト。Monaco不採用（重い） |
| 一時保存 | IndexedDB（Dexie.js） | WebView内。タブ内容の主保存先 |
| Vault書き出し | Tauri FS（Rust側） | `_drafts/`（二重保存バックアップ）と `inbox/`（昇格先）へ `.md` |
| AI | Claude API | あのあれ検索・AIタグ。`lib/claude.ts` に隔離 |

**このプロジェクトに存在しないもの（Helm 等の他プロジェクトと混同しないこと）**:
- Firebase / LiveKit / クラウド同期 → **一切使わない。完全ローカル**
- PDF 表示・マルチデバイス lease・リアルタイム購読 → **Perch には無関係**
- サーバー → **無い。デスクトップアプリ単体で完結**

## 🔴 絶対ルール

### セキュリティ
- Claude API キーは **コードに直書き禁止**。設定（ユーザー入力）or 環境変数経由で保持する。
- API キーを含む設定ファイル・`.env` 系は **Git にコミットしない**（`.gitignore` で除外）。
- Vault への書き込みはユーザーが選択したフォルダ配下（`inbox/` `_drafts/`）に限定する。想定外パスへの書き込み禁止。

### Claude API（`lib/claude.ts` に隔離）
- **Claude API を叩くコードは `lib/claude.ts`（ラッパー）1 箇所に集約する。** あのあれ検索・AIタグは全てこのラッパー経由。呼び出し側にモデル名やエンドポイントを直書きしない。
- **モデル名は設定化**：既定 `claude-sonnet-5`（2026/07 時点）。設定画面（S-07）で変更可能にする。新モデル対応は設定変更のみで済む構造を維持する。
- 優先順位は 設定値 > 既定値。将来の環境変数上書きも許容する構造にする。
- `max_tokens` は必ず指定する。
- **過剰な汎用化はしない**：「全バージョン自動対応」やレスポンス構造の完全汎用パースは抽象化過剰でメンテ不能になるため行わない。ただしラッパー内部は「AIに投げる」抽象にとどめ、将来の別プロバイダ乗り換えに構造だけ備える（今は作り込まない）。
- API キー未設定時は AI 機能を無効化し、ルールベースタグ＋手動＋非AI機能で動作を継続する（アプリを落とさない）。

### データ堅牢性（揮発防止）【最重要・ブロッカー】
- タブ入力は **1文字ごとに IndexedDB へ自動保存**（保存ボタンを設けない）。
- **二重保存必須**：debounce（数秒）で `_drafts/` にも `.md` を自動書き出し。IndexedDB を主、Vault ファイルを副とする。
- 強制終了・再起動でタブが失われないこと。失われうる範囲は debounce 設計値以内に最小化する。
- タブ破棄時は IndexedDB と `_drafts/` の両方から確実に削除する。
- `_drafts/` 書き出し失敗時も IndexedDB 保存は継続し、入力を止めない（控えめに警告表示し設定へ誘導）。

### エディタ（CodeMirror 6）
- 素の textarea ではなく CodeMirror 6 を使う。
- 実装するのは「一時メモ運用に必要な分だけ」：フォントサイズ変更・HTML/JS/CSS ハイライト・インデント保持・行番号。
- **やらない**：IntelliSense・自動補完・デバッグ等のガチ IDE 機能（Monaco の領域・過剰）。本格コード編集は VS Code / Claude Code に任せる。

### タグ付け（mozu 方式ハイブリッド）
- 3 系統：①ルールベース（本文走査で機械的付与）②AI（confidence 付き提案）③手動。
- **confidence 閾値（既定 0.8）で自動確定と提案止まりを分岐**。閾値以上は確定タグへ、未満は提案止まり（手動承認待ち）。閾値は設定で変更可能。
- **YAML `tags:` に書き出すのは確定タグのみ。** 未承認の `suggestedTags` は Vault に書き出さない（検索ノイズ防止）。

### TypeScript
- `any` は禁止。`unknown` を使うか適切な型を定義する。
- タブのデータモデル（`Tab` 型：`tags` と `suggestedTags` を分離）を型で明示し、確定/提案の混同を型レベルで防ぐ。

### テスト（詳細は `docs/test-spec.md`）
- **実装と同時にテストを書く**（後付け禁止）。
- **テスト pass = 機能完成 ではない。** Wave 完了判定には以下を全て満たすこと:
  - 自動テスト全 pass
  - `npm run build`（および `cargo build`）がローカルで成功
  - **実際にアプリを起動して主要フローを触って動作確認**（特に揮発防止は再起動・強制終了を実挙動で確認）
  - `docs/test-spec.md` の該当 TC を満たす
- **揮発防止テスト（TC-100 番台、特に TC-103 強制終了復元）は最重要ブロッカー。** ここが落ちたらリリース不可。

### 偽装テスト禁止【Helm での実被害を踏まえた最重要規律】
- **実際には検証していないのに通過するテスト（fake test）を禁止する。** `expect(true).toBe(true)` 相当、モックで本体を空洞化して「通っただけ」のテストは不可。
- 各テストは「本当にその機能が動いた証拠」を検証すること:
  - Vault 書き出し系 → **実ファイルの存在と内容**を確認（DB 上だけで完結しない）
  - タグ YAML 出力系 → **生成された `.md` の中身**を読んで検証
  - 揮発系 → **実際に kill / 再起動相当の状態遷移**を経る
  - `execSync` / `grep` でファイル存在を確認するだけの spec を「テスト」と認めない。実際の動作と出力/副作用を assert する。
- （背景）別プロジェクト Helm で「E2E 全 pass 報告後に実機で致命バグ」「偽装 E2E」を複数回踏んだ。ファイル存在確認や 200 応答だけを assert し、ユーザーが見る結果を確認していなかったことが原因。同型事故を Perch で繰り返さない。

### 開発フロー
- **`git push` 前に `npm run build`（＋ `cargo build`）をローカルで実行し、通ってから push する。**
- コミットは論理単位で整理（1 コミット = 1 つの変更目的）。準備ファイル / 実装 / 設定は分ける。
- 詳細な Git 運用は **GIT-WORKFLOW.md** に従う。

### プレースホルダーの扱い
- 拡張スロット（イメージ結合・`[[関連リンク]]`提案・RAG 化・タブ検索フィルタ）は **差し込み口（無効状態の UI 枠）** として実装してよい。
- その際は無効であることが明確に分かる表示にし、`// TODO(拡張): 〜` コメントで対応予定を示す。
- MVP に含む機能（揮発しない賢いmi・CodeMirror・Vault昇格・タグ付け・あのあれ検索）は **プレースホルダー禁止**。完成させる。

## 🚀 実装の進め方（Wave 想定）

フェーズ分割・中間確認は最小限。一気に動く形まで走らせ、自動チェック→合格なら次 Wave へ進む（要件定義書 §9 準拠）。

1. **Wave 1**: Tauri 2 プロジェクト雛形（macOS）＋ IndexedDB（Dexie）でタブ CRUD ＋自動保存 ＋再起動復元
2. **Wave 2**: CodeMirror 6 組込み（フォントサイズ・HTML/JS/CSS ハイライト）＋タブ一覧 UI ＋ピン留め
3. **Wave 3**: Vault 二重保存（Tauri FS・`_drafts/` 自動書き出し）＋ `inbox/` への `.md` 昇格（タイトル自動・YAML）
4. **Wave 4**: タグ付け（ルールベース）＋あのあれ検索（Claude API 都度方式・`lib/claude.ts`）
5. **Wave 5**: AI 自動タグ（confidence 自動確定＋手動確定 UI）＋拡張スロットの差し込み口＋仕上げ

各 Wave 完了時、`docs/test-spec.md` の該当 TC を実行し合格を確認する。Wave 1 完了時点で **揮発防止（TC-102/103/104）を最優先で検証**すること。
