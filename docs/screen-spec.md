## 2. S-01 メインウィンドウ（全体レイアウト）

```
┌──────────────┬───────────────────────────────────┐
│ タブ一覧       │  エディタ                          │
│ (S-02)        │  (S-03)                           │
│               │                                   │
│  🔍［タブ絞込］ │  ┌─ ツールバー ──────────────────┐ │
│               │  │ A- A+ | #タグ表示 | ［これ残す］ │ │
│  📌 タブA      │  └──────────────────────────────┘ │
│  📌 タブB      │  ┌──────────────────────────────┐ │
│  ・ タブC      │  │                              │ │
│  ・ タブD      │  │  (CodeMirror 6 本文)          │ │
│  ・ …          │  │                              │ │
│               │  │                              │ │
│  ［＋新規］     │  └──────────────────────────────┘ │
├──────────────┴───────────────────────────────────┤
│ 🔍 あのあれ検索：「____________________」 ［検索］     │
│   → 結果リスト（クリックでジャンプ）                  │
└───────────────────────────────────────────────────┘

```

- **分割比**：左ペイン 約25〜30%、右ペイン 残り。境界はドラッグでリサイズ可（状態は保存）。
- **下部検索バー**：常時表示。高さは結果0件時は1行、結果表示時に展開。
- **状態の永続化**：ペイン幅・最後に開いていたタブ・フォントサイズはIndexedDBに保存し、再起動時に復元。

## 3. S-02 タブ一覧ペイン

### 3.1 構成要素
| 要素 | 内容 | 挙動 |
|------|------|------|
| タブ絞込ボックス | タブ一覧内の簡易フィルタ | 入力でタイトル部分一致フィルタ（MVPは任意・拡張スロット扱いでも可） |
| ピン留めセクション | `pinned=true` のタブ | 一覧上部に固定表示。📌アイコン |
| 通常セクション | `pinned=false` のタブ | `updatedAt` 降順 |
| タブ項目 | タイトル（本文1行目由来） | クリックで S-03 に読込。昇格済みは軽い印（✓など） |
| ＋新規ボタン | 新規タブ作成 | 空タブを作成し即エディタにフォーカス |

### 3.2 タブ項目の操作
- **クリック**：エディタに開く（アクティブ表示）。
- **右クリック / ⋯メニュー**：ピン留めトグル / 破棄 / （昇格済みなら）Vaultで開く。
- **ピン留めトグル**：`pinned` を反転。一覧内で即セクション移動。
- **破棄**：未昇格タブは S-06 確認モーダル経由。破棄時に `_drafts/` の対応ファイルも削除。

### 3.3 表示ルール
- タイトルは本文1行目を自動採用。1行目が空なら「無題」＋作成時刻。
- タブが増えても破綻しないよう、一覧はスクロール可能。
- ※色分け・検索・並べ替え強化は拡張スロット（要件 §3.2）。

## 4. S-03 エディタペイン

### 4.1 ツールバー
| 要素 | 機能 | 備考 |
|------|------|------|
| A- / A+ | フォントサイズ縮小／拡大 | `Cmd -` / `Cmd +` も対応。設定値は永続化 |
| #タグ表示 | 現在のタブに付いている確定タグを表示 | 昇格前は「（昇格時に付与）」表示でも可 |
| ［これ残す］ | Vault昇格。S-05を開く | 昇格済みタブでは「更新」表示 |

### 4.2 本文エディタ（CodeMirror 6）
- Markdown編集。HTML/JS/CSSのシンタックスハイライト対応。
- インデント保持・行番号表示。
- **1文字打つごとにIndexedDBへ自動保存**（保存操作不要・保存ボタンなし）。
- debounce（数秒）で `_drafts/` にも `.md` 自動書き出し（二重保存）。
- 保存状態インジケータは**出さない**方針（常時保存済みが前提。ただし書き出し失敗時のみ警告を表示）。

### 4.3 エラー時挙動
- `_drafts/` 書き出し失敗（Vaultパス未設定・権限エラー等）→ ツールバーに控えめな警告アイコン。クリックで設定(S-07)へ誘導。IndexedDB保存は継続しているため入力は止めない。

## 5. S-04 あのあれ検索バー／結果

### 5.1 構成
| 要素 | 内容 |
|------|------|
| 検索入力 | 自然文クエリ（例「税金の話どっかでしたよね」） |
| ［検索］ボタン | Enterでも実行 |
| 結果リスト | 意味的に近いメモを順位付け表示。各行：タイトル＋抜粋＋所在（タブ or Vault） |

### 5.2 挙動
- 検索実行中はローディング表示（Claude API通信）。
- 結果クリック：タブ内メモならエディタで開く。Vaultノートなら該当ファイルを開く（Obsidianで開く or アプリ内表示はMVPでは「パス提示＋Obsidianで開く」で可）。
- オフライン時：検索は不可。「ネット接続が必要」と控えめに表示（他機能は動作継続）。
- 0件時：「近いメモが見つからなかった」を表示。

## 6. S-05 Vault昇格ダイアログ（mozu方式タグ確定）

```
┌────────────────────────────────────────────┐
│ Vaultに残す                              [×] │
├────────────────────────────────────────────┤
│ タイトル：[ 自動生成タイトル（編集可）      ] │
│ 保存先　：inbox/ （固定表示）                 │
│                                            │
│ 確定タグ：                                   │
│   #code/html  #2026-06  #税金 ✓   （×で削除）│ ← ルールベース＋AI自動確定
│                                            │
│ 提案タグ（承認待ち）：                        │
│   #確定申告  [承認][却下]                     │ ← confidence < 閾値
│   #経費      [承認][却下]                     │
│                                            │
│ ＋タグ追加：[ ____________ ]                  │
│                                            │
│              [ キャンセル ]  [ 昇格する ]     │
└────────────────────────────────────────────┘
```

### 6.1 タグの3系統（要件 §6.5準拠）
| 系統 | 由来 | 初期状態 | 表示位置 |
|------|------|----------|----------|
| ルールベース | 本文走査（```html/URL/日付等） | 確定 | 確定タグ欄 |
| AI自動確定 | Claude API `confidence >= 閾値` | 確定 | 確定タグ欄 |
| AI提案 | Claude API `confidence < 閾値` | 未確定 | 提案タグ欄 |
| 手動追加 | ユーザー入力 | 確定 | 確定タグ欄 |

### 6.2 挙動
- ダイアログを開いた時点でルールベースタグは即表示。AIタグは通信後に追記（ローディング表示）。
- 提案タグ [承認]→確定タグ欄へ移動。[却下]→消える。
- 確定タグの ×→削除。
- [昇格する]：`inbox/YYYY-MM-DD-slug.md` を書き出し。**確定タグのみ** YAML `tags:` に記載。`suggestedTags`（未承認）はVaultに書き出さない。
- 昇格後：タブに `promoted=true` / `promotedPath` を記録。既に昇格済みなら上書き確認。
- AI通信失敗時：ルールベース＋手動タグのみで昇格可能（AI欄に「取得失敗・スキップ可」表示）。

## 7. S-06 タブ破棄確認モーダル

- 未昇格タブの破棄時のみ表示（昇格済みはVaultに実体があるため確認省略も可）。
- 文言例：「このメモを破棄します。元に戻せません。」［破棄］［キャンセル］。
- 破棄実行：IndexedDBから削除＋`_drafts/`対応ファイル削除。

---

## 8. S-07 設定モーダル

| 項目 | 内容 | 既定 |
|------|------|------|
| Vaultパス | 昇格先 `inbox/`・バックアップ `_drafts/` の親フォルダ | 未設定（初回選択） |
| フォントサイズ既定 | エディタ初期サイズ | 中 |
| confidence閾値 | AIタグ自動確定のしきい値（0〜1） | 0.8 |
| 使用モデル | Claude APIのモデル名（`lib/claude.ts`が参照） | claude-sonnet-5 |
| Claude APIキー | あのあれ検索／AIタグ用 | 未設定 |
| （表示）バージョン等 | — | — |

- Vaultパス変更時は既存の `_drafts/` の扱い（移行 or 新規）を確認。
- APIキー未設定時：AI機能（検索・AIタグ）は無効化し、ルールベース＋手動のみで動作。

## 9. S-08 初回起動オンボーディング

- 初回のみ表示。目的：Vaultフォルダの選択。
- ステップ：①ようこそ ②Vaultフォルダ選択（必須） ③（任意）APIキー入力 → 完了。
- APIキーはスキップ可（後から設定可能）。Vault選択は必須。

## 10. 共通仕様

### 10.1 キーボードショートカット（案）
| キー | 動作 |
|------|------|
| `Cmd + N` | 新規タブ |
| `Cmd + +` / `Cmd + -` | フォントサイズ増減 |
| `Cmd + S` | 明示昇格（S-05を開く）※自動保存済みだが心理的安心用 |
| `Cmd + F` | あのあれ検索にフォーカス |
| `Cmd + ,` | 設定 |

### 10.2 状態の永続化対象
- 全タブ内容（IndexedDB主・`_drafts/`副）
- ペイン幅、最後のアクティブタブ、フォントサイズ、confidence閾値、Vaultパス、APIキー

### 10.3 デザイントークン（確定・墨と和紙）
```css
--bg:      #1a1a1e;   /* 墨 */
--fg:      #e8e6e1;   /* 生成り */
--accent:  #7c9885;   /* 苔緑 */
--muted:   #6b6b73;   /* 補助テキスト */
--danger:  #c17b6f;   /* 朱鷺色（破棄・警告） */
--panel:   #232329;   /* タブ一覧・ダイアログの面 */
```
- ダーク基調（常駐向き・目に優しい）。ライトモードは将来トークンで追加可能。
