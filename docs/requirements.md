## 1. 背景・課題

### 1.1 現状の使い方
- Mac上のテキストエディタ **mi** を一時メモ置き場として使用。タブを大量に開き、必要なものだけ保存、不要なものは破棄する運用。
- 確定メモは Google Keep に手入力 or mi でまとめてから Keep に転記。
- 散在により「探せない」「過去メモと繋がらない」課題がある。

### 1.2 核心的な痛点（最優先で解決すべきもの）
1. **揮発の恐怖**：mi は保存しない限り内容が揮発する。PC再起動（特に予期しない強制再起動）で未保存メモが消え、毎回サルベージ作業が発生している。
2. **探せない**：複数ツール（mi / Keep / 各種生成AI）に情報が散らばり、キーワードが曖昧だと見つけられない。
3. **繋がらない・増幅できない**：過去メモとの関連や、複数メモの結合による発想の増幅ができていない。

### 1.3 このツールの立ち位置
- **mi を置き換えるのではなく、「揮発しない賢い mi」として再現する。**
- 現在の「タブ大量展開→必要なものだけ残す」運用をそのまま踏襲しつつ、揮発を撲滅する。
- 残すと決めたメモは Obsidian Vault に `.md` 昇格して蓄積・検索・リンクの土台に乗せる。

## 2. 設計思想（重要）

> **「賢いmiは完成品、AIは拡張スロット」**

- **土台（賢いmi）は最初から100点で作る。** ここがチープだと土台ごと使われなくなる。妥協なし。
- **AI機能は"後回し"ではなく"差し込める設計"。** データ構造を最初からAI前提で整え、UIに差し込み口を用意する。
- **「あのあれ検索」はMVPに含める。** ユーザーが最も欲しがっている機能であり、都度API方式なら工数も軽い。チープにしない。
- **重いAI機能（イメージ結合・RAG化）は綺麗な差し込み口だけ用意し、実運用が見えてから後付けする。**

## 4. 技術スタック

| 項目 | 採用 | 理由 |
|------|------|------|
| アプリ形態 | **Tauri 2（デスクトップアプリ）** | Chromeと別プロセスで軽量常駐。Electronは中身がChromiumで重く（80〜200MB/メモリ120MB+）、"Chromeが重いから逃げたい"目的と矛盾するため不採用。TauriはOSのWebView（Mac=WebKit）利用で2〜10MB/メモリ30〜50MB |
| フロントエンド | Next.js（静的書き出し）or Vite + React | Tauri内のWebViewで描画。既存資産を流用 |
| バックエンド | Rust（Tauri標準・薄く使う） | ファイルアクセス等のOS機能のみ。ロジックの大半はJS/TS側に置き、Rustは最小限 |
| エディタ | **CodeMirror 6** | 軽量（基本〜50kB）・初期化ほぼ一瞬・フォントサイズ／コードハイライトが標準で揃う。Monacoは過剰（2〜5MBで重い）なので不採用 |
| 一時保存 | IndexedDB（Dexie.js推奨） | WebView内で使用可。永続・大容量・kotohogiで実績あり |
| Vault書き出し | **Tauriのファイルシステムアクセス（Rust側）** | ブラウザのFile System Access APIの制約（起動ごと再許可・Chromium限定）から解放。正規のファイルアクセスで二重保存が堅牢に |
| AI | Claude API（既定 `claude-sonnet-5`・設定で変更可） | あのあれ検索／AIタグ。`lib/claude.ts` ラッパーに隔離し、モデル名は設定化（§11.4）。Adjuvox構成流用 |
| 対象OS | **macOS**（当面） | ユーザー環境がMac。WebView差異の懸念なし |

## 5. データモデル

### 5.1 タブ（一時メモ）
```ts
interface Tab {
  id: string;            // uuid
  title: string;         // 本文1行目から自動生成（編集可）
  body: string;          // Markdown本文
  pinned: boolean;       // ピン留め
  createdAt: number;     // epoch ms
  updatedAt: number;     // epoch ms（1文字ごとに更新）
  tags: string[];        // 確定タグ（自動確定＋手動で確定したもの）
  suggestedTags?: {      // 未確定のAI提案タグ（承認待ち）
    name: string;
    confidence: number;  // 0〜1。閾値以上は自動確定、未満は提案止まり
  }[];
  promoted: boolean;     // Vault昇格済みか
  promotedPath?: string; // 昇格先の.mdパス
}
```
- **AI前提でフィールドを最初から綺麗に持つ**（後でAIに食わせる際の精度に直結するため、ここは妥協しない）。
- `tags` と `suggestedTags` を分離することで「確定済み」と「提案中」を明確に区別する（mozu方式の手動確定を支える構造）。

### 5.2 Vault昇格時の出力（.md）
```markdown
---
title: "（自動生成タイトル）"
created: 2026-06-29T14:30:00+09:00
source: perch
tags: [code/html, link]   # ルールベース＋AI自動確定タグが入る
---

（本文）
```
- ファイル名：`YYYY-MM-DD-（タイトルをslug化）.md`
- 既定の昇格先フォルダ：Vault内 `inbox/`（迷わせない。整理はObsidian側で後から）

## 6. 機能要件

### 6.1 揮発しない賢いmi【最優先・妥協なし】
- 新規タブをワンアクションで作成、いくつでも開ける。
- 本文入力中、**1文字打つたびに自動でIndexedDBへ保存**（保存ボタン不要）。
- **二重保存**：IndexedDBを主とし、一定間隔（例：数秒のdebounce）でVault内 `_drafts/` にも各タブを `.md` で自動書き出し（バックアップ）。**Tauriのファイルシステムアクセス（Rust側）で正規に書き込む**ため、ブラウザのFile System Access APIのような起動ごとの再許可が不要で堅牢。ブラウザデータが飛んでもMac上にファイルが残る。
- **常駐性**：Tauriの独立ウィンドウとして常駐。Chromeを閉じても・Chromeが重くても影響を受けない。「常にそこにある賢いmi」を実現。
- PC／ブラウザ再起動後、**全タブがそのまま復元**される。
- 不要タブは明示的に破棄（メール下書きのように気軽に捨てられる）。破棄時は `_drafts/` の対応ファイルも削除。

### 6.2 エディタ仕様（CodeMirror 6）【一時メモ運用に必要な分だけ】
- **本体**：CodeMirror 6 を採用。素のtextareaではなく、軽量で拡張性の高いエディタを土台にする。
- **フォントサイズ変更**：`Cmd +` / `Cmd -` または UI ボタンで拡大縮小。設定はIndexedDBに保持し再起動後も維持。
- **コードのシンタックスハイライト**：HTML / JS / CSS を対象（言語パッケージを追加）。Claude Codeが生成したコード片を一時バックアップする用途に対応。
- **インデント保持・行番号**：標準機能として有効化。
- **やらないこと（割り切り）**：IntelliSense・自動補完・デバッグ等のガチIDE機能は載せない（Monacoの領域・過剰）。本格的なコード編集はVS Code / Claude Codeに任せる。本ツールはあくまで「一時バックアップ＋メモ」。

### 6.3 タブ一覧・操作
- 左ペインにタブ一覧（タイトル表示）。
- **ピン留め**：よく使うタブを上部固定。
- 更新日時順ソート（デフォルト）。
- ※検索・色分けは拡張スロット（タブが増えてから後付け）。

### 6.4 Vault昇格
- 「これ残す」ボタンで、現在のタブを Vault `inbox/` に `.md` 昇格。
- タイトル自動付与（本文1行目ベース）＋YAMLフロントマター付与。
- 昇格後、タブには昇格済みフラグ＋パスを記録（重複昇格を防ぐ・上書き確認）。

### 6.5 タグ付け（mozu方式ハイブリッド）【MVPに含む】
ルールベース・AI・手動の三層で、mozuと同じく「自動で付けつつ手動で最終調整」できる形にする。

**① ルールベース自動タグ（軽い・オフライン・Claude API不要）**
- 本文を機械的に走査して即付与。書いた瞬間／昇格時に動く。
- ルール例（実装時に調整）：
  - ` ```html ` を含む → `#code/html`、` ```js ` → `#code/js`、` ```css ` → `#code/css`
  - URLを含む → `#link`
  - 昇格日から `#YYYY-MM`（例 `#2026-06`）
- 高速・確実。ノイズになりにくい機械的なタグを担当。

**② AI自動タグ（賢い・Claude API・要ネット）**
- 昇格時に本文をClaude APIに渡し、意味的なタグ（例 `#税金` `#確定申告`）を **confidence（信頼度 0〜1）付き** で提案。
- 「あのあれ検索」で作るClaude API基盤を流用（追加工数を抑える）。
- **confidence閾値による自動確定**（楽さ優先の要望に対応しつつノイズを防ぐ安全装置）：
  - `confidence >= 閾値`（例 0.8）→ **自動確定**して `tags` に入れる
  - `confidence < 閾値` → `suggestedTags` に入れ、**提案止まり**（手動承認待ち）
  - 閾値は設定で変更可能。

**③ 手動タグ（mozu方式・最終確定）**
- ①②で付いたタグを、昇格ダイアログで人間が承認・追加・削除して確定。
- 提案タグはワンクリックで承認／却下。手動追加も可能。
- **確定タグ（`tags`）だけがYAMLフロントマターに書き出される。** 提案止まり（`suggestedTags`）はVaultに漏らさない（検索ノイズ防止）。

### 6.6 あのあれ検索【MVPに含む・チープにしない】
- フワッとした自然文クエリ（例：「税金の話どっかでしたよね」）を入力。
- **対象**：全タブ本文 ＋ Vault内 `.md`（読み込み可能な範囲）。
- **方式（MVP）**：都度方式。関連候補をまとめて Claude API に送り、意味的に近いメモを順位付けして返す。
- 結果はクリックで該当タブ／ノートへジャンプ。
- ※メモが大量化したら RAG（ベクトル検索）へ進化（拡張スロット）。

### 6.7 拡張スロット（差し込み口を用意）
- AI検索パネルに「関連メモをあぶり出す」「選択した複数タブをまとめて増幅する」ボタンの**場所だけ用意**（初期は無効 or プレースホルダ）。
- 昇格ダイアログに、AIによる `[[関連リンク]]` 提案欄の**枠だけ用意**（AIタグ提案自体は6.5でMVP実装済み）。

## 7. 非機能要件
- **動作の軽快さ**：賢いmi部分はサクサク動くこと（入力遅延・保存遅延を感じさせない）。土台の快適さは最優先。
- **オフライン動作**：AI検索以外はネット不要で完結。デスクトップアプリとしてローカルで動作。
- **データ堅牢性**：揮発防止が本ツールの存在意義。IndexedDB＋Vault二重化を必須要件とする。
- **プライバシー**：メモ本体はローカル完結。AI検索時のみ必要な範囲を Claude API に送信（送信範囲は最小化）。

## 9. 実装の進め方（Wave想定）
- フェーズ分割・中間確認は最小限。一気に動く形まで走らせる。
- 自動チェック→合格なら次Waveへ。
- 想定Wave（目安）：
  1. Tauri 2プロジェクト雛形（macOS）＋IndexedDB（Dexie）でタブCRUD＋自動保存＋再起動復元

  5. AI自動タグ（confidence自動確定＋手動確定UI）＋拡張スロットの差し込み口（無効状態のUI枠）＋仕上げ

## 10. 受け入れ基準（MVP完了の定義）
- [ ] タブを10個以上開いて入力し、ブラウザ／PC再起動後に全タブが復元される。
- [ ] 未保存状態が存在しない（常時自動保存されている）ことを確認できる。
- [ ] 任意のタブを Vault `inbox/` に `.md` 昇格でき、Obsidianで開ける。
- [ ] 昇格時、ルールベースタグ（`#code/html` 等）が自動付与される。
- [ ] AIが提案したタグのうち、confidence閾値以上は自動確定し、未満は提案止まり（手動承認待ち）で表示される。
- [ ] 提案タグを手動で承認／却下／追加でき、確定タグだけがYAMLに書き出される。
- [ ] `_drafts/` にバックアップ `.md` が自動生成され、タブ破棄で消える。
- [ ] あのあれ検索に曖昧な自然文を入れると、意味的に近いメモが返り、ジャンプできる。
- [ ] 賢いmi部分の操作にストレスがない（妥協なしの土台になっている）。

## 11. 決定事項

### 11.1 ツール名
- **Perch（パーチ）**。「止まり木」の意。メモが飛び立つ（Vault昇格）前に一時的にとまる場所。既存アプリ `Mozu`（百舌鳥）と鳥の世界観で連続性を持たせた。揮発しない一時メモというコンセプトにも合致。

### 11.2 配色（ダーク基調「墨と和紙」）
一日中常駐する用途のため目が疲れないダーク基調。真っ黒の冷たさを避け、和の温かみを持たせる。`kokage`（凪）/`kotohogi` のトーンと地続き。

```css
--bg:      #1a1a1e;   /* 墨。わずかに紫みのある背景 */
--fg:      #e8e6e1;   /* 生成り色。真っ白を避け目に優しい */
--accent:  #7c9885;   /* くすんだ苔緑。和の落ち着き */
--muted:   #6b6b73;   /* 補助テキスト */
--danger:  #c17b6f;   /* 破棄・警告。朱鷺色（赤すぎない） */
--panel:   #232329;   /* タブ一覧・ダイアログの面 */
```
- コンセプト：「墨と和紙」（背景=墨、文字=生成りの紙、アクセント=苔）。
- ライトモードは将来追加できる構造にしておく（トークンで管理）。

### 11.3 Vaultパス
- 設定画面（S-07）で選択。初回起動オンボーディング（S-08）で**必須選択**とする。
- 選択した親フォルダ配下に、昇格先 `inbox/` とバックアップ `_drafts/` を配置。

### 11.4 Claude APIのメンテナンス性
- **モデル名を設定化**：設定画面（S-07）に「使用モデル」欄を追加。デフォルト `claude-sonnet-5`。新モデル対応時は設定変更のみで済む。
- **API処理を1ファイルに隔離**：Claude APIを叩くコードは `lib/claude.ts`（ラッパー）に集約。あのあれ検索・AIタグは全てこのラッパー経由で呼ぶ。モデル変更もAPI仕様変更も、修正箇所をここ1点に限定する。
- **優先順位**：設定値 > デフォルト値。将来的な環境変数上書きも許容する構造にする。
- **過剰な汎用化はしない**：「全バージョン自動対応」やレスポンス構造の完全汎用パースは、抽象化過剰でメンテ不能になるため行わない。ただしラッパー内部は「AIに投げる」抽象にしておき、将来の別プロバイダ乗り換えに構造だけ備える（今は作り込まない）。
